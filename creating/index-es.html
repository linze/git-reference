Obtención y creación de proyectos

Antes de hacer algo con Git, tienes que tener a mano un repositorio Git. Esto es donde Git almacena la información de las instantáneas del proyecto que estás guardando.

Hay dos formas principales de obtener un repositorio Git. Una de ellas consiste básicamente en inicializar (crear) uno nuevo a partir de un directorio ya existente, como un nuevo proyecto o un proyecto ya existente al que se le quiere hacer control del código fuente. La segunda manera consiste en clonar un repositorio Git público, tal como harías si quisieras una copia del proyecto o fueras a trabajar con alguien. Cubriremos esas dos formas a continuación.

documentos   libro git init inicializa un directorio como repositorio Git

Para crear un repositorio a partir de un directorio ya existente, puedes ejecutar simplemente git init en ese directorio. Por ejemplo, supongamos que tenemos un directorio con unos cuantos ficheros en él, como esto: 
$ cd konichiwa
$ ls
README   hello.rb
Esto es un proyecto donde estamos escribiendo ejemplos del programa "Hello World" en cada lenguaje de programación. Por ahora tenemos solo el ejemplo en Ruby, pero bueno, es un comienzo. Para comenzar a controlar las versiones con Git, podemos ejecutar git init. 
$ git init
Initialized empty Git repository in /opt/konichiwa/.git/
Ahora podrás ver que hay un subdirectorio llamado .git en tu proyecto. Este es tu repositorio Git donde toda la información de las instantáneas de tu proyecto son almacenadas. 
$ ls -a
.        ..       .git     README   hello.rb
Felicidades, ahora tienes la base de tu repositorio Git y puedes empezar a crear las instantáneas de tu proyecto.

En resumen, usar git init para hacer que un directorio existente se convierta en un repositorio Git. Puedes hacer esto en cualquier directorio, en cualquier momento, de forma totalmente local

documentos   libro git clone copia un repositorio Git existente en el cual puedes trabajar

Si necesitas colaborar con alguien en un proyecto, o si quieres obtener una copia del proyecto de forma que puedes leer y/o usar el código, lo clonarás. Simplemente ejecuta la orden git clone [url] especificando la URL del proyecto que quieres copiar. 

$ git clone git://github.com/schacon/simplegit.git
Initialized empty Git repository in /private/tmp/simplegit/.git/
remote: Counting objects: 100, done.
remote: Compressing objects: 100% (86/86), done.
remote: Total 100 (delta 35), reused 0 (delta 0)
Receiving objects: 100% (100/100), 9.51 KiB, done.
Resolving deltas: 100% (35/35), done.
$ cd simplegit/
$ ls
README   Rakefile lib

Esto copiará todo el historial del proyecto asi que lo tendrás localmente y te dará un directorio de trabajo de la rama principal de ese proyecto, de forma que puedes mirar el código o comenzar a editarlo. Si cambias al nuevo directorio, puedes ver el subdirectorio .git - ahí es donde está toda la información del proyecto. 

$ ls -a
.        ..       .git     README   Rakefile lib
$ cd .git
$ ls
HEAD        description info        packed-refs
branches    hooks       logs        refs
config      index       objects

Por defecto, Git creará un directorio con el mismo nombre que el proyecto que le pasas por la URL - básicamente será lo que sea que esté escrito despues de la última barra de la URL. Si quieres algo distinto, escríbelo al final del comando, después de la URL.

En resumen, usas git clone para obtener una copia local de un repositorio Git de forma que puedas observarlo o empezar a trabajar en él.
